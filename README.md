# dentisti.pro - Multi-Tenant Dental Clinic Management System

A complete, production-ready PHP + MySQL multi-tenant SaaS application for dental clinics. Built for shared hosting (Hostinger/cPanel/Apache) with zero dependencies.

## Features

### Multi-Tenant Architecture
- Super admin platform management
- Multiple independent dental clinics
- Per-clinic user management and data isolation
- Flexible subscription plans (Starter, Pro, Enterprise)

### Core Dental Features
- **Patient Management**: Complete patient records with medical history, allergies, insurance
- **Dental Tooth Chart**: FDI numbering system (11-48) with visual tooth status tracking
- **Appointments**: Calendar view, scheduling, status tracking, reminders log
- **Treatments**: Clinical notes, procedures, diagnosis, file attachments
- **Billing**: Invoice generation, payment tracking, multiple payment methods
- **Inventory**: Material tracking with low-stock alerts

### Modern 2025 UI/UX
- Clean, minimal clinical design
- Card-based dashboard with real-time metrics
- Responsive mobile-first layout
- Generous spacing (24-32px)
- Rounded corners (12-16px)
- Status badges and color coding
- Sticky topbar with breadcrumbs

### Bilingual Support
- Albanian (sq) and English (en)
- Language switcher in topbar
- All UI strings translated
- Per-clinic language settings

### Security
- Password hashing (bcrypt)
- CSRF token protection
- Activity logging
- Session management
- Role-based access control (Admin, Doctor, Reception)

## Installation

### Requirements
- PHP 8.0 or higher
- MySQL 5.7+ or MariaDB 10.3+
- Apache with mod_rewrite
- Extensions: mysqli, mbstring, gd

### Steps

1. **Upload Files**
   - Upload all files to your web hosting
   - Ensure `config/` and `public/uploads/` are writable (755)

2. **Run Installer**
   - Navigate to `http://yourdomain.com/install/installer.php`
   - Follow the 5-step installation wizard:
     1. System requirements check
     2. Database configuration
     3. Write config file
     4. Import database schema
     5. Create super admin account

3. **Access Admin Panel**
   - Super Admin: `http://yourdomain.com/admin/login.php`
   - Create your first clinic from the admin panel

4. **Clinic Login**
   - Staff Login: `http://yourdomain.com/login.php`
   - Use credentials created by super admin

## Default Credentials

After installation, use the super admin credentials you created during setup.

## Super Admin Features

- Create and manage clinics
- Assign subscription plans
- Set per-clinic limits (users, patients, storage)
- Manage landing page content (bilingual)
- View platform-wide statistics
- Configure integrations (Stripe, SMS, SMTP - placeholders)

## Clinic Roles

### Admin
- Full access to all clinic features
- User management
- Clinic settings
- Reports

### Doctor
- View assigned patients
- Manage appointments
- Add treatments and clinical notes
- Update tooth chart
- Issue invoices (if enabled)

### Reception
- Create patient records
- Book appointments
- Issue invoices
- Check-in patients

## Customization

### Branding
- Logo: Upload via clinic settings
- Colors: Edit CSS variables in `public/assets/css/style.css`
- Landing page: Manage content via super admin panel

### Language
- Add new languages: Create new file in `lang/` directory
- Update `Lang.php` to support additional languages

### Plans
- Modify plans via super admin panel
- Adjust features, limits, and pricing

## Support

For issues or questions:
1. Check file permissions (config/, uploads/)
2. Verify PHP version and extensions
3. Review Apache error logs
4. Ensure database credentials are correct
